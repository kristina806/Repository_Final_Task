Git

  • git init - инициализирует новый пустой Git-репозиторий в текущей директории.
  • git clone (URL) - копирует существующий репозиторий из удаленного источника (например, GitHub) на твой компьютер.
  • git status - показывает текущий статус рабочего каталога и индекса, то есть какие изменения подготовлены для коммита, какие файлы не отслеживаются и т.д.
  • git add (файл/папка) - добавляет изменения в индекс (стейджинг), подготавливая их к последующему коммиту. Чтобы добавить все изменения, можно использовать git add .
  • git commit -m "сообщение" - фиксирует изменения, добавленные через git add, в репозиторий. Сообщение коммита описывает изменения.
  • git push - отправляет коммиты из локального репозитория в удаленный репозиторий.
  • git pull - обновляет локальный репозиторий, подтягивая изменения из удаленного репозитория.
  • git branch - показывает список веток или создает, удаляет ветки.
  • git checkout <ветка> - переключается на указанную ветку.
  • git merge <ветка> - сливает изменения из указанной ветки в текущую ветку.
